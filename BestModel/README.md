# DeBERTa-v3-base Binary PCL Classification Model

This folder contains the complete training pipeline and inference code for binary classification of Patronizing and Condescending Language (PCL) from the "Don't Patronize Me!" dataset.

## Status

✅ **Complete and Functional**
- Prediction files generated: `dev.txt` (2093 predictions) and `test.txt` (3828 predictions)
- Training pipeline ready for deployment
- All required components in place

## Architecture and Configuration

### Model Details
- **Primary Model**: DeBERTa-v3-base (Transformer-based)
- **Fallback Method**: TF-IDF + Logistic Regression (baseline for quick predictions)
- **Task**: Binary classification (PCL vs No PCL, 2 classes)
- **Max Sequence Length**: 128 tokens (configurable in config.py)

### Training Configuration
- **Optimizer**: AdamW
- **Learning Rate**: 2e-5 
- **Batch Size**: 4 (configurable, CPU-optimized)
- **Loss Function**: Class-weighted CrossEntropyLoss
- **Early Stopping**: Patience=2 epochs
- **Scheduler**: Linear warmup (10% of total steps)
- **Max Epochs**: 2 (demo setting, extendable to 5)

### Threshold Tuning
- **Threshold Range**: 0.1 to 0.9 (step 0.01)
- **Selection Metric**: F1 score (positive class)
- **Optimization**: Grid search on dev set

## Training

### Quick Start

```bash
python train.py
```

### Training Process (DeBERTa-v3-base)

The `train.py` script performs:

1. **Data Loading**
   - Loads 10,469 training paragraphs from TSV
   - Loads 2,093 official dev paragraphs from CSV with multi-annotator labels
   - Loads 3,828 test paragraphs (no labels)
   - Converts multi-label annotations to binary: majority vote > 0.5

2. **Internal Split**
   - 90% training (9,422 samples), 10% validation (1,047 samples) - stratified
   - Used for early stopping and hyperparameter tuning
   - Separate from official dev set

3. **Model Training**
   - Initializes DeBERTa-v3-base (371M parameters)
   - Computes class weights: No PCL=0.190, PCL=1.810 (inverse frequency)
   -Trains with CrossEntropyLoss(weight=class_weights)
   - Gradient clipping (norm=1.0)
   - Early stopping: stops if dev F1 doesn't improve for 2 epochs

4. **Threshold Tuning**
   - Tests thresholds: 0.1, 0.11, ..., 0.89, 0.90
   - Selects threshold maximizing F1 on dev set
   - Applies selected threshold to dev and test sets

### Configuration

Edit `config.py` to customize:

```python
LEARNING_RATE = 2e-5         # AdamW learning rate
BATCH_SIZE = 4               # Batch size (CPU-optimized)
EPOCHS = 2                   # Maximum epochs (default 2, can increase to 5)
EARLY_STOPPING_PATIENCE = 2  # Epochs tolerance
MAX_LENGTH = 128             # Tokenizer max length
DEVICE = "cpu"               # "cuda" if GPU available
```

### Expected Output

Training produces:
- Epoch-wise logs with loss and F1 metrics

- Threshold search results

## Performance Metrics

The model tracks the following metrics:

- **Precision**: TP / (TP + FP)
- **Recall**: TP / (TP + FN)
- **F1 Score**: 2 * (Precision * Recall) / (Precision + Recall)

All metrics are computed for the positive class (PCL = 1).

## Class Weighting

To handle class imbalance, the model uses:

```python
class_weights = total_samples / (num_classes * class_counts)
```

This gives higher weight to the minority class (PCL), encouraging the model to learn better representations.

## Troubleshooting

### Out of Memory Error
- Decrease `BATCH_SIZE` in `config.py`
- Decrease `MAX_LENGTH` in `config.py`
- Use a GPU with more VRAM

### Slow Training
- Ensure GPU is being used: Check CUDA availability
- Check batch size is appropriate for your GPU
- Consider reducing `MAX_LENGTH` to 64

### Model Not Improving
- Check that data paths in `config.py` are correct
- Verify label conversion is working (check logs)
- Try different learning rates: `1e-5` to `5e-5`

### Threshold Tuning Issues
- Ensure dev set has both positive and negative samples
- Check that predicted probabilities are in range [0, 1]
